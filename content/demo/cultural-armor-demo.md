# Cultural Armor Infographic Demo

This page demonstrates the Cultural Armor infographic shortcode - an interactive visualization explaining how Oxymiron built a three-layer protection system against censorship through strategic cultural encoding.

## Concept Overview

The **Cultural Armor** system represents how Oxymiron created immunity against censorship through:

1. **Layer 1: Literary Foundation** - Grounding work in classical Russian poetry traditions
2. **Layer 2: Visual Encoding** - Multi-layered meaning through cryptographic approaches
3. **Layer 3: Title Surgery** - Strategic abstraction while preserving core message

## Interactive Features

- **Click any layer** to reveal detailed analysis
- **Hover effects** showing layer relationships
- **Attack visualization** demonstrating censorship attempts
- **Mobile-responsive** with touch gesture support
- **Keyboard navigation** (1-3 for layers, 0/C for core, Esc to reset)
- **Accessibility compliant** with WCAG 2.1 AA standards

The infographic features:

1. **SVG-based armor visualization** with translucent layered shields
2. **Interactive layer reveals** with detailed information panels
3. **Attack simulation** showing censorship attempts deflected
4. **Gradient effects and filters** creating depth and visual hierarchy

## Design Philosophy

### Visual Metaphor
- **Medieval/futuristic shield design** representing protection
- **Translucent overlapping layers** showing system architecture
- **Color coding** (gold, silver, steel) for different protection levels
- **Attack arrows** visualizing censorship attempts being deflected

### Typography
- **Source Serif Pro** for literary quotes and Russian text
- **Inter** for technical labels and interface elements
- **Proper contrast ratios** meeting WCAG 2.1 AA standards

### Interaction Design
- **Progressive disclosure** revealing complexity on demand
- **Intuitive visual feedback** for all interactive elements
- **Mobile-first approach** with touch-optimized interactions
- **Keyboard accessibility** for non-mouse users
